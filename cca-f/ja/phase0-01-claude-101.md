### レッスン 01: What is Claude?

**URL:** <https://anthropic.skilljar.com/claude-101/383389>  

所要時間の目安: 15分

**学習目標**

**このレッスンを終えると、以下ができるようになります:**

Claude とは何か、そしてその設計を導く原則を説明できる
Claude の中核的な能力と、単純なチャットボットとの違いを説明できる
Claude にアクセスするさまざまな方法（Web、デスクトップ、モバイル）を識別できる
（10分）

Claude は単なるチャットボット以上の存在であり、あなたの思考のパートナーとなるよう設計された AI アシスタントです。このレッスンでは、Claude を他の AI ツールと差別化する要素を学び、さまざまな業務タスクにどう役立つかを見ていきます。

**重要なポイント**

Claude は helpful（役に立つ）・harmless（無害である）・honest（誠実である）であるように作られています: 高いレベルで言えば、Claude は有害または差別的な出力を避け、人間が違法または非倫理的な活動に関与するのを手助けせず、広く役に立ち、誠実で、無害な AI システムとして振る舞うのに役立つ原則によって導かれています。Constitutional AI と呼ばれるこのアプローチは、Claude が人間の価値観に整合し、透明性をもって動作するよう訓練されていることを意味します。
Claude はチャットボット以上の存在です: Claude は、要約、検索、創造的・協働的なライティング、Q&A、コーディングなど、幅広い会話やテキスト処理のタスクを、高い信頼性と予測可能性を保ちながらこなすことができます。Claude を、単純な質問に答えるだけでなく、複雑な問題に取り組み、困難な状況を一緒に乗り越えてくれる思考のパートナーとして捉えてください。
Claude は操縦可能（steerable）で協働的であるよう設計されています: Claude は人格、トーン、振る舞いに関する指示を受け取ることができ、顧客からは、Claude は有害な出力を生み出す可能性がはるかに低く、対話しやすく、より操縦しやすいと報告されています。そのため、より少ない労力で望む出力を得ることができます。
Claude にはあなたが働く場所どこからでもアクセスできます: Claude アプリはすべてのプランタイプ（Free、Pro、Max、Team、Enterprise）で利用可能です。サインインしていれば、会話、プロジェクト、メモリ、設定がすべてのデバイス間で同期されます。デスクにいても外出先にいても、Claude は Web、デスクトップ、モバイルアプリを通じて利用できます。
Claude の能力を理解する

Claude は、単純な一問一答のやり取りをはるかに超え、あなたの業務を自動化し拡張できるアシスタントのようなパートナーシップにまで及ぶ、幅広いタスクを手助けできます。Claude が得意とすることをいくつか挙げます:

ライティングとコンテンツ作成: Claude は、ソーシャルメディアの投稿、プロフェッショナルなメール、複雑なレポートをあなたと協働して作成できます。Claude は人格やトーンに関する指示を受け取るよう訓練されているため、あなたの声が明確に伝わるまで、構成や明瞭さを一緒に反復改善できます。
リサーチと分析: Claude は、リサーチの切り口を探り、調査結果をまとめ、データを分析して意味のある洞察を浮かび上がらせるのを手助けします。ドキュメントをアップロードすれば、Claude が複雑な情報を理解する手助けをします。これは Claude の大きなコンテキストウィンドウによって可能になっており、20万トークン以上（テキストで約500ページ以上に相当）を取り込むことができ、Opus 4.7 を使用する場合は Pro、Max、Team、Enterprise プランで最大100万トークンまで利用可能です。これにより Claude は、1回の会話の中で膨大な資料を考慮することができます。
コーディング支援: Claude Opus 4.7 は当社のこれまでで最も強力なモデルであり、世界最高のコーディングモデルです。実世界のコーディングタスクにおけるこの優れた性能により、Claude は複数のプログラミング言語にわたってコードを書き、デバッグし、説明する手助けができます。
問題解決と推論: Claude は、複雑な認知タスク、数学の問題、戦略的思考と分析、リサーチを扱えます。Claude Opus 4.7 と Sonnet 4.7 はハイブリッドモデルであり、ほぼ即時の応答と、より深い推論のための拡張思考（extended thinking）という2つのモードを提供します。Anthropic の拡張思考により、Claude は問題を段階的に解いていくことができ、慎重な分析を必要とするタスクに適しています。
新しいことを学ぶ: 新しいスキルを学んでいるとき、馴染みのない分野を探求しているとき、あるいは複雑な課題に取り組んでいるときでも、Claude はあなたの学習スタイルとペースに適応できます。Learning mode は新しい Claude の体験であり、答えを与えるのではなくあなたの推論プロセスを導き、批判的思考スキルの発達を助けます。

claude.com のユースケースギャラリーを探索して、あなた特有の職務で Claude を活用する方法のヒントを得てください。AI に何ができて何ができないかをさらに深く知りたい場合は、当社の AI Capabilities コースをご覧ください。

Claude にアクセスする方法

Claude はインテリジェンス（知性）です。すなわち、このコースを通じてあなたが協働の仕方を学んでいる AI アシスタントです。その同じ知性が複数のインターフェース

### レッスン 02: Your first conversation with Claude

**URL:** <https://anthropic.skilljar.com/claude-101/383390>  

所要時間の目安: 20分

**学習目標**

このレッスンを終えると、以下ができるようになります:

Claude との新しい会話を開始し、インターフェースをナビゲートできる
明確で具体的な言葉を使って効果的なプロンプトを書ける
ファイルや画像をアップロードして Claude に追加のコンテキストを与えられる
フォローアップメッセージを使って Claude の応答を反復・改善できる
動画: Getting started with Claude.ai

**重要なポイント**

Claude は強力で知的な協働者であり、あなたのすべての業務にわたって能力を増幅します。Claude は AI の知性をもたらしますが、その仕事を意味あるものにするコンテキストと専門知識をもたらすのはあなたです。
Claude に話しかける際の最良のアプローチは、同僚に話しかけるときのように、自然に、簡潔に、会話的に行うことです。
Claude との次の会話の前に、次のことを検討しましょう: 状況設定（あなたの役割、目的、コンテキスト）、タスクの定義（Claude に取ってほしい行動）、ルールの指定（スタイル、トーン、例）。
関連するドキュメントや背景情報をチャットにアップロードすると、Claude はその内容を応答に反映します。Claude があなたのニーズを理解するための近道だと考えてください。
Claude の真の力は、一度きりのプロンプトではなく、継続的で頻繁なコミュニケーションによって発揮されます。
最初の会話を始める

Claude.ai を開くと、画面下部にテキスト入力エリアのあるすっきりしたインターフェースが表示されます。

プロンプトは、（新機能のコードネームをブレインストーミングするような）単純な質問から、ファイルを共同作成する複雑なリクエストまで、さまざまです。

効果的なプロンプトを書く

Claude とのすべてのやり取りはプロンプトから始まり、これらのプロンプトは他のコンテキストと組み合わさって Claude の応答に影響します。Claude に話しかける際の最良のアプローチは、同僚に話しかけるときのように、自然に、簡潔に、会話的に行うことです。

しかし、良いプロンプトとは何か、と疑問に思うかもしれません。Claude との次の会話の前に、いくつかのことを検討しましょう:

状況設定: あなたの役割は何で、目的は何ですか? あなたの業務について Claude が知っておくべきコンテキストはありますか?
タスクの定義: Claude にどんな行動を取ってほしいですか? Claude に書いてほしいのか、分析してほしいのか、構築してほしいのか、それとも何か別のことですか?
ルールの指定: Claude に使ってほしいスタイルやトーンは何ですか? あなたが求めているものを Claude に示すために添付できる例はありますか?
まとめてみる

以下は、3つの要素すべてを使ったプロンプトの例です:

「私はインディーストリーミングのスタートアップでマーケティングリードを務めており、シリーズ A 投資家向けの投資家ピッチデックを準備しています。独立系映画ストリーミング市場の現状を調査し、主要なトレンド、競合のポジショニング、成長機会を特定してもらえますか? 最新の Web リサーチを引用付きで使い、エグゼクティブサマリー、市場分析、競合状況、成長機会を含む最大5ページのプロフェッショナルなレポートとして構成してください。」

このプロンプトでは:

状況設定: これが新しいインディーストリーミングアプリの投資家ピッチデックのためのものであることを Claude に伝えています。これがコンテキストと目的です。
タスクの定義: 具体的な行動（市場の調査）を、関連する詳細（トレンド、競合、機会）とともに提供しています。
ルールの指定: 引用付きの最新 Web リサーチを依頼し、プロフェッショナルなレポートとして構成するよう求めています。すなわち、必要なスタイルとフォーマットを Claude に正確に伝えています。

さらに深く知りたいですか? このプロンプトのフレームワークは、Rick Dakan 教授（Ringling College of Art and Design）と Joseph Feller 教授（University College Cork）の研究協力を通じて開発された AI Fluency のための 4D フレームワークを応用したものです。このフレームワークは、AI との効果的な協働を可能にする4つの中核的なコンピテンシー（Delegation、Description、Discernment、Diligence）を特定しています。

当社の無料の AI Fluency コースを受講して、4つのコンピテンシーすべてがどのように連携して、効率的・効果的・倫理的・安全な AI 利用を可能にするかを学んでください。

コンテキストを追加する

アップロード、コネクタ、カスタム設定（preferences）は、あなたの業務についてさらに多くのコンテキストを Claude に与える方法を提供します。

Claude は、PDF やその他のドキュメント内のテキストと視覚的要素（画像、チャート、グラフィックなど）の両方を分析できます。サポートされているファイルタイプには、PDF、DOCX、CSV、TXT のほか、PNG や JPEG などの一般的な画像フォーマットが含まれます。

ファイルアップロードを使う実用的な方法をいくつか挙げます:

ドキュメントをアップロードし、Claude に要点を要約するよう依頼する
画像を共有し、Claude が見ているものを説明または分析するよう依頼する
スプレッドシートを添付し、Claude にデータのトレンドを特定するよう依頼する
コードをアップロードし、Claude にその仕組みを説明したりバグを見つけたりするよう依頼する

アップロードすると、Claude は自動的にファイルの内容を解析しようとします。チャット内ではファイルが添付ファイルとして表示され、それについて Claude にプロンプトを送ることができます。

プロのヒント: すべての応答で Claude に特定の設定を考慮してほしい場合は、Settings > General > 「What personal preferences should Claude consider?」に移動して、すべての会話に適用される設定を行ってください。

### レッスン 03: Getting better results

**URL:** <https://anthropic.skilljar.com/claude-101/383392>  

所要時間の目安: 15分

**学習目標**

このレッスンを終えると、以下ができるようになります:

AI を使い始めたときによくある課題を認識し、それを克服するためのトラブルシューティング手法を使える
AI Fluency を定義し、AI と流暢に協働する方法をさらに学ぶための場所を知る
あなた独自のワークフローで Claude がどのように機能しうるかをよりよく理解するために、どのように評価（evals）を設定しうるかを説明できる
よくある課題とその解決方法

Claude を使い始めると、応答が期待したものと少し違う場面に遭遇することがよくあります。これは正常なことであり、アプローチを洗練させる機会でもあります。ここでは、最もよくある課題のいくつかと、その対処方法を紹介します。

課題 何が起きているか 試してみること
Claude の応答が一般的すぎる プロンプトにあなたの具体的な状況についての十分なコンテキストが含まれていなかった オーディエンス、役割、制約についての詳細を追加しましょう。「プロジェクトの遅延についてのメールを書いて」ではなく、「ソフトウェア統合が2週間遅れることをエンタープライズ顧客に説明するメールを書いてください。彼らはこれまで辛抱強く待ってくれていましたが、これが2回目の遅延です。プロフェッショナルでありながら謝罪の気持ちを込めてください。」と試してみましょう。
応答が長すぎる（または短すぎる） Claude が適切な長さを推測している 明示しましょう: 「2段落の要約をください」または「100語以内に収めてください」または「包括的な分析が必要です。長さは気にしません。」
Claude が私のフォーマットに従わなかった Claude はあなたが何を望んでいるかは理解したが、どのように提示してほしいかは理解していなかった 伝えるだけでなく、示しましょう。フォーマットの例を提供するか、構造を明示的に説明しましょう: 「各セクションに太字の見出しを付けた箇条書きを使ってください。」
自信ありげな情報を得たが、結局それが間違っていた Claude は、特に具体的な事実やニッチなトピックについて、もっともらしいが不正確な情報を生成することが時折あります 重要度の高い業務では、主要な事実を独立して検証しましょう。Claude に出典を引用させるか、確信度を示すよう依頼しましょう。Web 検索を有効にして、応答を最新の情報に基づかせましょう。
トーンが適切でない Claude はデフォルトで役に立ちプロフェッショナルな調子になりますが、それがあなたのニーズに合わないことがあります トーンを平易な言葉で説明しましょう: 「もっと会話的にしてください」または「これは権威的でフォーマルな調子にすべきです。」あなたが望むスタイルの文章の例を提供しましょう。
反復のマインドセット

Claude と協働する際の最も重要な転換の1つは、最初のプロンプトが完璧な結果を生み出すことはめったにないと認識することです。そして、それでよいのです。最初のプロンプトは、一発勝負のリクエストではなく、会話の始まりだと考えましょう。

効果的な Claude ユーザーは:

最初のドラフトを出発点として扱います。Claude が生み出したものをレビューし、何がうまくいっていて何がうまくいっていないかを特定し、それから洗練させます。
具体的なフィードバックを与えます。「もっと短く」でも構いませんが、「最初の2段落を削って、結論をもっと行動志向にしてください」のほうが優れています。
ゼロからやり直すべきタイミングを知っています。会話が脱線してしまった場合、軌道修正しようとするよりも、より明確なプロンプトで新しいチャットを開くほうが早いこともあります。
AI Fluency とは?

AI Fluency とは、AI ツールと効果的に協働する能力のことです。単にどのボタンをクリックすればよいかを知っているだけでなく、さまざまな状況にわたって AI をうまく使う判断力を養うことです。

AI Fluency のための 4D フレームワークは、Rick Dakan 教授（Ringling College of Art and Design）と Joseph Feller 教授（University College Cork）の研究協力を通じて開発されたもので、組み合わせることで AI とのやり取りを最大限に活用できる4つの中核的なコンピテンシーを特定しています:

Delegation（委任）: どの作業を人間が行うべきか、どの作業を AI が行うべきか、そしてそれらの間でどのようにタスクを分配するかを決めること。自分の目標と AI の能力を理解し、協働について戦略的な選択を行うことを含みます。
Description（記述）: AI システムと効果的にコミュニケーションすること。出力を明確に定義し、AI のプロセスを導き、望ましい AI の振る舞いやインタラクションを指定することを含みます。
Discernment（識別）: AI の出力、プロセス、振る舞い、インタラクションを思慮深く批判的に評価すること。品質、正確性、適切性を評価し、改善すべき領域を見極めることを含みます。
Diligence（誠実な遂行）: AI を責任を持って倫理的に使うこと。AI システムやインタラクションについて思慮深い選択を行い、透明性を保ち、AI 支援による成果物に対して説明責任を負うことを含みます。

あなたはこのコースを通じて、すでにこれらのスキルを実践してきました。レッスン2のプロンプトフレームワーク（状況設定、タスクの定義、ルールの指定）は Description に根ざしています。上記のトラブルシューティング手法は Discernment と Diligence を活用しています。

さらに詳しく知りたい場合は、4つのコンピテンシーすべてを深く掘り下げ、実践的な演習や実世界での応用を扱う当社の無料の AI Fluency コースをご覧ください。

あなたのワークフローのために Claude を評価する

Claude をより多くの業務に統合し始めると、こう疑問に思うかもしれません: この特定のタスクで Claude が本当に優れているかどうかを、どうやって知ればよいのか?

ここで Discernment が不可欠になります。Evals（evaluations の略）は、Claude の出力をタスクごとに評価する直感を養うための方法であり、それらのタスク

### レッスン 04: Claude desktop app: Chat, Cowork, Code

**URL:** <https://anthropic.skilljar.com/claude-101/440908>  

学習内容

所要時間の目安: 6分

**このレッスンを終えると、以下ができるようになります:**

Claude デスクトップアプリの3つのモード（Chat、Cowork、Code）と、それぞれが何のために設計されているかを識別できる
クイックエントリー、スケジュールタスク、ローカル開発とリモート開発の違いなど、各モードに固有の主要機能を説明できる
達成したい業務の種類に基づいて、適切なモードを選択できる
Claude デスクトップアプリをナビゲートする: Chat、Cowork、Code

Claude デスクトップアプリは、Claude と協働する3つの方法（Chat、Cowork、Code）を提供します。簡単な質問から、複雑なリサーチ、ソフトウェアの構築まで対応します。

Chat は claude.ai でおなじみの同じ Claude に加え、コンピュータ上でネイティブに動作することから得られるクイックエントリー、スクリーンショット、ディクテーション、コネクタが利用できます。Cowork はエージェント型のツールです。目標を与え、ツールやリソースに接続して、作業を任せます。Cowork では、Claude はより多くのことを行うためのリーチと余地を持ちます。この広い範囲により、より徹底的なリサーチや分析を行い、より複雑なドキュメントや成果物を生み出すことができます。Code は、コードの作成やテストからデプロイまで、ソフトウェアを構築するためのものです。

Cowork と Code は同じエンジン上で動作します。両方とも内部的には Claude Code であり、あなたのマシン上でローカルに動作し、自律的に作業でき、サブエージェントを立ち上げ、長時間のタスクを維持できます。これにより Claude は、リサーチとライティングやソフトウェアの構築のような、より大きなタスクを自力でこなすことができます。

各モードは、それが扱う業務を中心に設計されており、重要なものを見せ、必要なところで制御を与えます。

Chat

Chat は、質問したり、ブレインストーミングしたり、ドラフトを作成したり、行き来しながら問題を解決したりする必要があるときに真価を発揮します。

claude.ai を使ったことがあれば、これは同じように機能しますが、コンピュータ上でネイティブに動作することから得られるいくつかの点があります:

クイックエントリー。Mac で Option キーをダブルタップすると、作業中のものの上に Claude を呼び出せます。Claude はコンパクトなウィンドウで応答し、アプリを切り替えても最前面に留まります。質問するために、今やっていることを離れる必要は一切ありません。
スクリーンショットとウィンドウ共有。スクリーンショットをキャプチャしたりウィンドウを共有したりすれば、Claude はあなたが見ているものとまったく同じものを見られます。画面に何があるかを説明するより速く、より正確です。（Mac）
ディクテーション。タイプする代わりに、問題を声に出して話せます。考えを口に出しているとき、キーボードから離れているとき、または話すほうが書くより速い場面で役立ちます。（Mac）
デスクトップコネクタ。コネクタを通じてローカルのツールやサービスに接続すれば、Claude はあなたのマシン上の他のツールと連携できます。

次のようなときに試してみましょう:

馴染みのないダッシュボードを前にしているとき。Option をダブルタップし、ウィンドウ上でカーソルをドラッグしてスクリーンショットを撮り、「これらのメトリクスは何を意味しているの?」と尋ねます。Claude はダッシュボードを表示したまま、オーバーレイで答えます。
会議と会議の合間に、プレゼンテーションの構成を考えたいとき。クイックエントリーを開き、音声に切り替えて、話しながら考えを整理します。Claude はあなたが話した内容からアウトラインのドラフトを作成します。

**学習目標**

このレッスンを終えると、以下ができるようになります:

名前、説明、可視性設定を付けて新しいプロジェクトを作成できる

Claude の振る舞いを導く効果的なプロジェクト指示を書ける

動画

**重要なポイント**

プロジェクトは、独自のメモリ、チャット履歴、ナレッジベース、カスタマイズされた指示を持つ、自己完結型のワークスペースです。特定の業務ストリームのための専用環境だと考えてください。
プロジェクトナレッジは、関連するドキュメントをアップロードすることで Claude の理解を高めます。Claude はそのプロジェクト内のすべてのチャットでそれらを参照します。毎回同じファイルを再アップロードする必要はもうありません。
プロジェクト指示は Claude の振る舞いを導きます。トーン、専門性のレベル、応答スタイルなどを指定できます。これらの指示はプロジェクト内のすべての会話に適用されます。
プロジェクトは自動的にスケールします。ナレッジベースがコンテキストの上限に近づくと、Claude はシームレスに Retrieval Augmented Generation（RAG）モードを有効にし、応答品質を保ちながら容量を最大10倍まで拡張します。
Claude for Work ユーザー向けに、プロジェクトはコラボレーションを可能にします。チームメイトとプロジェクトを共有すれば、全員が同じコンテキスト、指示、蓄積されたナレッジの恩恵を受けられます。
プロジェクトとは?

プロジェクトは、Claude が参照すべきナレッジを保管したり、特定のトピックや業務領域を中心に関連するチャットを整理したり、同じ共有コンテキストへのアクセスを必要とするチームメンバーと協働したりするのに最適です。

プロジェクトを使うべきとき

プロジェクトは、一度きりの質問ではなく、継続的な何かに取り組んでいるときに特に価値があります。次のようなワークフローがある場合に、プロジェクトの作成を検討しましょう:

繰り返し使う参考資料（会議メモ、調査結果、レポート、過去のデータなど）
Claude がどう応答すべきかについての一貫した要件（常にフォーマルな言葉を使う、常に出典を引用する、常に当社のテンプレートに従う）
複数の人が同じ基盤から作業すべきチームコラボレーションのニーズ
最初のプロジェクトを作成する

プロジェクトのセットアップは数分しかかかりません。始め方は次のとおりです:

ステップ1: プロジェクトをセットアップする
左サイドバーにカーソルを合わせて「Projects」をクリックするか、claude.ai/projects に直接移動します
右上隅の「+ New Project」をクリックします
プロジェクトにわかりやすい名前を付けます（例:「Q4 Marketing Campaign」または「Product Documentation」）
取り組んでいる内容の簡単な説明を追加します。Claude はこの説明を直接見るわけではありませんが、あなたとチームメイトがプロジェクトの目的を理解するのに役立ちます。
可視性設定を選びます: 非公開にするか、組織と共有するか（Claude for Work ユーザー向け）
ステップ2: プロジェクト指示を追加する

プロジェクト指示は、このプロジェクト内のすべての会話にわたって Claude にどう振る舞うべきかを伝えます。「Instructions」をクリックして指示パネルを開きます。

良いプロジェクト指示には通常、次のものが含まれます:

取り組んでいる内容についてのコンテキスト:「このプロジェクトは、当社の B2B ソフトウェア製品のマーケティングコンテンツを作成するためのものです。」
プロセスの指示:「まずこのオーディエンスを引き込むブログ構成を検討し、それからドラフトを書いてください。」
トーンとスタイルの好み:「プロフェッショナルだが会話的なトーンを使ってください。可能な限り専門用語を避けてください。」
具体的な要件:「マーケティングコピーの最後には必ず行動喚起（call-to-action）を含めてください。」

指示を書き終えたら、「Save instructions」をクリックします。これらはこのプロジェクト内のすべてのチャットに適用され、設定済みのユーザー設定やスタイルと併せて機能します。

プロジェクト指示を使ってワークフローを自動化することもできます。たとえば「会議の文字起こしをアップロードしたら、このテンプレートを使って構造化された要約を作成してください」のように。指示は、このプロジェクトのための Claude の振る舞いをプログラミングするものだと考えてください。

ステップ3: ナレッジベースを構築する

プロジェクトのナレッジベースは、Claude が参照すべきドキュメントをアップロードする場所です。ファイルメニューはプロジェクトのメインページの右側にあります。

「+」ボタンをクリックしてコンテンツを追加します。PDF、DOCX、CSV、TXT、HTML などさまざまなファイルタイプをアップロードできます。Google Drive に接続してドキュメントを直接リンクすることもできます。

アップロードすべきもの:

背景資料（リサーチレポート、会議メモ、要件ドキュメント）

アーティファクトとは?

アーティファクトは、Claude が会話と並んだ専用ウィンドウに作成する、独立したインタラクティブな出力です。長いコードやテキストのブロックがチャットに埋もれてしまう代わりに、コンテンツがレンダリングされ、すぐに使える状態で表示されます。それが動作する Web サイトであれ、インタラクティブなチャートであれ、すぐにダウンロードできるドキュメントであれです。

Claude は、コンテンツが特定の条件を満たすと自動的にアーティファクトを作成します:

それが重要で自己完結している（通常15行以上）
あなたが編集したり、反復改善したり、再利用したくなりそうなものである
周囲の会話を必要とせず単独で成立する複雑なコンテンツを表している
後で参照したり使ったりしたくなるコンテンツである
よくあるアーティファクトのタイプ

Claude はさまざまなアーティファクトを作成でき、それぞれが異なるニーズに適しています:

ドキュメント（Markdown、プレーンテキスト、Word ドキュメント、PDF、PowerPoint、Excel を含む）: エクスポートしたり編集を続けたりしたくなるテキスト中心のあらゆるものに最適です。会議メモ、レポート、プロジェクト計画、ブログ投稿、その他の文章コンテンツなどです。
コードスニペット: あらゆるプログラミング言語（Python、JavaScript、C++ など）で動作するコード。コードを閲覧したり、コピーしたり、ダウンロードして自分のプロジェクトで使ったりできます。
HTML ページ: HTML、CSS、JavaScript を1つのファイルにまとめた完全な Web ページ。ランディングページ、フォーム、インタラクティブなデモ、簡単なプロトタイプに最適です。
SVG 画像: ロゴ、アイコン、イラスト、その他の視覚的要素のためのスケーラブルベクターグラフィックス。これらはアーティファクトウィンドウ内で直接レンダリングされるため、得られるものを正確に確認できます。
Mermaid ダイアグラム: フローチャート、シーケンス図、ガントチャート、組織図など。視覚化したい関係を説明すれば、Claude が改善できるダイアグラムを作成します。
React コンポーネント: 実際の機能を持つインタラクティブな UI 要素。電卓、ダッシュボード、ゲーム、データ可視化などです。これらは単なるモックアップではなく、実際のロジックを含み、ユーザー入力に応答します。

「当社の顧客オンボーディングプロセスを示すフローチャートを作成してください（注: Claude は、コードベースのアーティファクトに加えて、Imagine を使ってフローチャートのような視覚的ダイアグラムを HTML として生成するようになりました。）」
「月々の支出を入力して内訳を確認できるインタラクティブなダッシュボードを構築してください」
「ヒーローセクションと機能リストを備えた、生産性アプリのランディングページをデザインしてください」
「新しい取り組みに再利用できるプロジェクトブリーフのテンプレートを書いてください」

期待したときに Claude が自動的にアーティファクトを作成しない場合は、明示的に依頼できます:「これをアーティファクトとして作成して」または「これをアーティファクトで見せて」。

### レッスン 07: Working with skills

**URL:** <https://anthropic.skilljar.com/claude-101/383396>  

所要時間の目安: 15分

**学習目標**

このレッスンを終えると、以下ができるようになります:

Skills（スキル）とは何か、Claude がそれをどう使うかを説明できる
ドキュメント作成のための Anthropic 組み込みスキルを識別できる
設定でスキルを有効化し管理できる
プランごとの提供状況: Skills は現在、Pro、Max、Team、Enterprise プラン向けの機能プレビューです。Free プランの場合は、概念を理解するために読み進め、ハンズオンの手順はスキップできます。
Skills とは?

Skills は、Claude が専門的なタスクのパフォーマンスを向上させるために動的に読み込む、指示・スクリプト・リソースのフォルダーです。専門知識のパッケージだと考えてください。特定のタスクを再現可能な方法で完了する方法を Claude に教えます。

Claude を使って Excel スプレッドシート、PowerPoint プレゼンテーション、Word ドキュメント、PDF を作成したことがあれば、すでにスキルが動作しているのを目にしています。それらのファイル作成機能は、舞台裏で動作するスキルによって支えられています。しかしスキルはドキュメント作成をはるかに超えます。カスタムスキルは、四半期ごとの差異分析の方法論、ブランドボイスのレビュープロセス、コンプライアンスチェックリストなど、繰り返し可能なワークフロー全体を体系化でき、Claude が毎回同じ厳格な手順に従うようにできます。

スキルのタイプ

あなたが出会うスキルには2つのカテゴリーがあります:

Anthropic スキルは、Anthropic が作成し維持しているものです。これには、Excel、Word、PowerPoint、PDF ファイルのための強化されたドキュメント作成機能が含まれます。Anthropic スキルはすべての有料ユーザーが利用でき、関連する場合に Claude が自動的に呼び出します。使うために特別なことをする必要はありません。
カスタムスキルは、あなたや組織が専門的なワークフローやドメイン固有のタスクのために作成するものです。たとえば、自社のブランドガイドラインをプレゼンテーションに適用するスキル、会議メモを特定のフォーマットで構造化するスキル、組織のデータ分析ワークフローを実行するスキルなどを作成できます。
スキルを有効化する

Skills は現在、Pro、Max、Team、Enterprise プランのユーザー向けの機能プレビューとして提供されています。スキルを使うには、Code execution and file creation（コード実行とファイル作成）を有効にする必要があります。スキルは機能するために Claude の安全なサンドボックス化されたコンピューティング環境を必要とするためです。

スキルを有効化する方法は次のとおりです:

Settings > Capabilities に移動します
Code execution and file creation がオンになっていることを確認します
Skills セクションまでスクロールします
必要に応じて個々のスキルをオンまたはオフに切り替えます

Enterprise プランの場合、個々のメンバーがスキルにアクセスできるようになる前に、組織の Owner がまず Admin 設定で Code execution と Skills の両方を有効にする必要があります。

Team プランの場合、この機能プレビューは組織レベルでデフォルトで有効になっています。

有効化すると、設定に利用可能なスキルが一覧表示されます。Anthropic の組み込みスキルや、アップロードしたカスタムスキルが含まれます。

実践でスキルを使う

スキルの素晴らしい点は、通常それらについて考える必要がないことです。Claude があなたのリクエストに基づいて自動的にスキルを選択します。スキルを呼び出すプロンプトの例をいくつか挙げます:

「合計の数式付きで月々の支出を追跡する Excel スプレッドシートを作成して」
「この会議メモのドキュメントを PowerPoint プレゼンテーションに変換して」
「このデータを要約した PDF レポートを生成して」
「シナリオ分析付きの財務モデルを Excel で構築して」

Claude がスキルを使うと、作業を進める際の Claude の思考の連鎖（chain of thought）の中でそれが言及されているのが見えます。出力は、コンピュータまたは Google Drive に直接保存できるダウンロード可能なファイルになります。

ファイル実行

Claude はスライド、スプレッドシート、契約書のレッドライン（修正履歴）であなたと協働します

この同じ機能により、Claude は（隔離された環境内で）あなたの実際のファイルを扱い、ファイルの更新版を作成できます（注: Chat では、Claude は元のドキュメントをその場で編集するのではなく、新しいバージョンのドキュメントを作成します）。スライド、スプレッドシート、契約書（または任意の .xlsx、.pptx、.docx、.pdf ファイル）をアップロードすれば、Claude がスライドを作成し、分析を行い、修正案を追加する様子を見られます。Claude が完了したら、これらのファイルをダウンロードするか、Drive で開けます。

注: これらの機能を使うには、Claude に外部データソースへのアクセスを与える必要があります。プロンプトが表示されたら、Allow limited network access（限定的なネットワークアクセスを許可する）をオンに切り替えるだけです:

セキュリティに関する考慮事項

スキルは実行可能なコードを含みうるため、思慮深く使うことが重要です:

信頼できるソースからのカスタムスキルのみをインストールしましょう
Anthropic の組み込みスキルは Anthropic によってテストされ維持されています
あなたがアップロードするカスタムスキルは、あなた個人のアカウントに非公開です
外部ソースからカスタムスキルをインストールする場合は、使用前にその内容をレビューして、何をするかを理解しましょう。
カスタムスキルを作成する

Anthropic の組み込みスキルは一般的なドキュメント作成タスクをカバーしますが、スキルの真の力は自分自身のスキルを作成することから生まれます。カスタムスキルを使えば、あなた特有のワークフロー、ブランドガイドライン、仕事のやり方を Claude に教えることができ、関連するときにはいつでも Claude がその知識を自動的に適用できます。

カスタムスキルを作成する最も簡単な方法は、Claude 自身との会話を通じて行うことです。

### レッスン 08: Connecting your tools

**URL:** <https://anthropic.skilljar.com/claude-101/383397>  

所要時間の目安: 20分

**学習目標**

このレッスンを終えると、以下ができるようになります:

コネクタとは何か、なぜそれが Claude との業務にとって重要なのかを説明できる
コネクタディレクトリをナビゲートし、最初の接続をセットアップできる
Claude との会話で接続済みのツールを効果的に使える
コネクタとは?

**重要なポイント**

コネクタは、あなたが毎日使っているのと同じツール、データ、コンテキストへのアクセスを Claude に与えることで、Claude をアシスタントから情報に通じた協働者へと変えます。すべての会話をゼロから始める代わりに、Claude はあなたの実際の情報を使って直接作業できます。
コネクタにより、Claude はあなたに代わって情報を読み、アクションを実行できます。コネクタと付与した権限に応じて、Claude はファイルを検索し、ドキュメントを取得し、データを分析し、新しいコンテンツを作成し、レコードを更新し、接続済みのアプリケーション全体でタスクを実行できます。すべて会話の中から行えます。
Model Context Protocol（MCP）がコネクタを支えています。MCP は AI のための USB-C のようなものだと考えてください。単一の一貫したインターフェースを通じて、Claude が多くの異なるアプリケーションに接続できる普遍的な標準です。このオープン標準は、開発者が任意のツール向けにコネクタを構築でき、それらのコネクタが Claude とシームレスに連携することを意味します。
コネクタには2種類あります: Web コネクタとデスクトップ拡張機能です。Web コネクタは、Google Drive、Notion、Slack、Asana などのクラウドサービスに Claude をリンクします。デスクトップ拡張機能は、Claude Desktop アプリを通じてコンピュータ上でローカルに動作し、ローカルファイルやネイティブアプリケーションへのアクセスを Claude に与えます。
ツールを見つけて接続する

Anthropic は、claude.ai/directory で推奨コネクタのディレクトリを維持しています。ディレクトリは2つのタブに整理されています:

Web: クラウドサービスとアプリケーション（Gmail、Notion、Slack、Asana、Linear、Stripe など多数）
Desktop extensions: Claude Desktop アプリを通じてコンピュータ上で動作するローカルツール

利用可能なコネクタを閲覧するには、チャットウィンドウの左下にある + ボタンをクリックし、Connectors を選択することもできます。

Web コネクタをセットアップする

クラウドサービスを接続する方法は次のとおりです:

コネクタを見つける: claude.ai/directory に移動するか、任意のチャットで + > Connectors をクリックします
Connect をクリック: 追加したいコネクタを選択します
認証する: サービスのログインページにリダイレクトされます。既存の資格情報でサインインします
権限を付与する: Claude が要求している具体的な権限をレビューし、アクセスを承認します
接続をテストする: Claude に戻り、「私の [ツール名] にアクセスできる?」のような簡単なリクエストを試します

接続すると、付与した権限に応じて、Claude はそのサービス内で検索したり、読み取ったり、場合によってはアクションを取ったりできます。

デスクトップ拡張機能

デスクトップ拡張機能は、Web インターフェースではなく Claude Desktop アプリを必要とします。これらの拡張機能により、Claude は macOS または Windows のローカルアプリケーション、ファイルシステム、ネイティブ機能とやり取りできます。

デスクトップ拡張機能には次のようなものがあります:

ドキュメントの読み取りと整理のためのローカルファイルアクセス
自動化された Web タスクのためのブラウザ制御
ネイティブアプリケーション統合（デザイン作業のための Figma など）

デスクトップ拡張機能をインストールするには:

Claude Desktop アプリをダウンロードしてインストールします
アプリを開き、Settings > Extensions に移動します
利用可能な拡張機能を閲覧し、Install をクリックします
その拡張機能に固有の追加のセットアップ手順に従います
業務でコネクタを使う

ツールを接続すると、Claude はあなたのリクエストに応答する際にそれらを考慮します。接続済みのツールを使う実用的な方法をいくつか挙げます:

プロジェクト管理（Asana、Linear、Jira）

「今週締め切りの最優先タスクは何?」
「Q4 予算案のレビューのための新しいタスクを作成して」
「当社のプロダクトローンチプロジェクトのステータスを要約して」

コミュニケーション（Slack、Gmail）

「ベンダー契約について話し合ったメールスレッドを見つけて」
「#marketing チャンネルの最新メッセージへの返信のドラフトを書いて」
「昨日の議論で、タイムラインについてチームは何を決めた?」

ドキュメンテーション（Notion、Google Drive、Confluence）

「当社のブランドボイスガイドラインをドキュメントから検索して」
「先週のプロダクトレビューの会議メモを要約して」
「当社のスタイルガイドは短縮形（contractions）の使用について何と言っている?」

ビジネスツール（Stripe、PayPal、Salesforce）

「過去四半期の収益トレンドを見せて」
「Acme Corp の商談のステータスは?」
「1,000ドルを超える最近の取引を一覧表示して」
セキュリティと権限

Claude を外部サービスに接続するとき、あなたはそれらのサービス内のデータを読み取る（場合によっては変更する）アクセス権を付与しています。いくつかの重要な考慮事項を挙げます:

スコープ付きアクセス: 権限はコネクタが必要とするものに固有であり、各アプリケーションのメニュー内で個々の権限をオンオフ切り替えできます。
Claude はあなたが見るものを見る: Claude はあなたがアクセスできるデータにのみアクセスできます。仕事用メールを接続しても、Claude が CEO の受信トレイにアクセスできるようになるわけではありません。あなた自身のものだけです。
いつでも取り消し可能: Claude の設定を通じて、またはサードパーティサービスの設定

### レッスン 09: Enterprise search

**URL:** <https://anthropic.skilljar.com/claude-101/383398>  

所要時間の目安: 15分

**学習目標**

このレッスンを終えると、以下ができるようになります:

Enterprise Search とは何か、Enterprise Search が答えられる質問の種類を説明できる
管理者とユーザーの両方にとってセットアッププロセスがどのように機能するかを理解する
セキュリティと権限が組織のデータをどう保護するかを認識する
プランごとの提供状況: Enterprise Search は Team および Enterprise プランで利用可能で、ワークスペース管理者が有効にする必要があります。Free、Pro、Max プランの場合は、このレッスンをスキップできます。
Enterprise Search とは?

Enterprise Search は、サイドバーに専用の「Ask {あなたの組織名}」オプションを追加します。これは、会社のツールやデータソース全体に埋もれているナレッジを見つけて統合するために特別に設計されています。Enterprise Search を、組織全体のための事前構築済みプロジェクトだと考えてください。会社のナレッジベースがすでに読み込まれているため、すぐに飛び込んで、質問に対するコンテキストに応じた応答を得られます。

コネクタが有効になった通常のチャットとは異なり、Enterprise Search は情報収集のために特別に設計されており、Anthropic チームによって構成されたカスタム指示を使用します。

何を尋ねられるか?

Enterprise Search は、複数のソースにまたがる質問や、組織全体の情報を統合する必要がある質問に特に価値があります。一般的なユースケースをいくつか挙げます:

最新状況を把握する

「私が不在の間、昨日何があった?」
「先週からのビジネス全体にわたる主要な更新を要約して」
「Platform プロジェクトの現在のブロッカーは何?」

ポリシーとプロセスに関する質問

「当社のリモートワークポリシーは何?」
「経費精算書はどう提出するの?」
「休暇申請のプロセスは?」

リサーチと分析

「顧客が競合他社を選ぶ際に挙げる主な理由は何?」
「Q4 プロダクトロードマップについての議論を要約して」
「当社の顧客オンボーディングプロセスについての情報を見つけて」

新しいチームメンバーのオンボーディング

「当社の認証システムはどう機能するの?」
「課金システムの学習について誰に相談すべき?」
「エンジニアリングチームはデプロイにどんなツールを使っている?」

パフォーマンスとプロジェクトの追跡

「マーケティングキャンペーンに関連する議論やドキュメントを見つけて」
「先週のリーダーシップ会議での主要な決定は何だった?」
「Infrastructure イニシアチブへのチームの貢献を要約して」

質問すると、Claude は接続済みのすべてのツール（SharePoint ドキュメント、Slack の会話、Gmail スレッド、Google Drive ファイルなど）を横断して検索し、情報を統合された応答にまとめます。さらに、常に出典を引用するため、全体のコンテキストを得られます。

Enterprise Search をセットアップする

Enterprise Search は2段階のセットアッププロセスを必要とします: まず管理者が組織のために構成し、それから個々のユーザーが個人アカウントで認証します。

管理者（Owners）向け

Enterprise Search プロジェクトは、すべての Team および Enterprise 組織でデフォルトで有効になっていますが、チームメンバーが使えるようになる前に Owner が初期セットアップを完了する必要があります:

左サイドバーで「Ask Your Org」をクリックします。
「Set up for your org」をクリックして続行します（または「Disable」で機能をオフにします）。
組織のツールを接続します。Documents（Google Drive や SharePoint など）と Chat（Slack や Microsoft Teams など）のコネクタを選択する必要があります。Email は推奨ですが任意です。
「+ Add more」をクリックして、チームが必要とする追加のツールをセットアップします。
プロジェクト名をカスタマイズします。入力した内容が、全員のサイドバーに「Ask [名前]」として表示されます。
説明を追加し、「Finish set up」をクリックします。

セットアップが完了すると、プロジェクトは組織のすべてのメンバーが利用できるようになります。

ユーザー向け

管理者が Enterprise Search をセットアップした後、「Ask {組織名}」プロジェクトがサイドバーにスター付きで表示されます。始め方は次のとおりです:

サイドバーのプロジェクトをクリックします。
ガイド付きのオンボーディングフローに従って、推奨サービスに接続します。
検索したい各サービス（Slack、Google、Microsoft 365 など）で認証します。
組織のナレッジについて Claude に質問し始めます。

有効にするコネクタが多いほど、検索結果はより包括的になります。プロジェクトの Instructions セクションで「Connect」をクリックすれば、後からいつでもコネクタを追加できます。

大量のデータだけど…これは安全?

端的に言えば、はい。Enterprise Search は、元の接続済みツールであなたがすでにアクセス権を持っているものだけを表示します。さらに、あなたの会話は非公開のままであり、接続されたデータが別途インデックスされたり保存されたりすることはありません。

### レッスン 10: Research mode for deep dives

**URL:** <https://anthropic.skilljar.com/claude-101/383399>  

所要時間の目安: 15分

**学習目標**

このレッスンを終えると、以下ができるようになります:

Research が何をするかを説明できる: 体系的で多ソースの調査
包括的な情報収集のために Research をいつ使うべきかを識別できる
Research が拡張思考（extended thinking）とどう連携して徹底的なレポートを提供するかを理解する
複雑な調査のための効果的な Research プロンプトを書ける
Claude でリサーチする

**重要なポイント**

Research は、Claude が情報を見つけて分析する方法を変えます。単一の検索ではなく、Claude はエージェント的に動作します。次に何を調査すべきかを正確に判断しながら、互いに積み重なる複数の検索を行います。あなたの質問のさまざまな切り口を自動的に探求し、未解決の問いを体系的に解決していきます。
Research は数分で包括的な答えを提供します。ほとんどのレポートは5〜15分で完了しますが、より複雑な調査は最大45分かかることもあります。通常なら手作業で何時間もかかる作業です。
拡張思考は Research で自動的に有効になります。この強力な組み合わせにより、Claude はアプローチを思慮深く計画し、包括的に情報を収集し、複雑なリクエストを扱いやすい部分に分解できます。
引用により検証が容易になります。Research は、確認しやすい引用付きの徹底的な答えを提供するため、Claude の調査結果を信頼でき、出典を自分ですばやく検証できます。
Research とは?

Research は、Claude を会話型アシスタントから体系的な調査者へと変える高度な機能です。Research を有効にすると、Claude は単に質問に答えるのではなく、複数の角度からそれを探求し、Web と接続済みの統合（integrations）にわたる情報を統合します。

何時間もかけて情報を収集し、ソースを相互参照し、包括的なレポートをまとめられる熟練したリサーチアシスタントがいるようなものだと考えてください。ただし、それが何時間もではなく数分で起こります。

Research は、簡単な答え以上のものが必要なときに特に価値があります。徹底的な理解のために複数のソースから情報をまとめ、異なる視点を比較し、調査結果を実行可能な洞察に統合する必要がある状況のために設計されています。

Research を使うべきとき

Research を他の Claude の機能と比べていつ使うべきかを理解すると、あなた特有のニーズに対して最良の結果を得られます。

次が必要なときに Research を使いましょう:

複数のソースから情報を統合する包括的なレポート
Web と接続済みの統合（Google Workspace など）にわたる詳細な分析
通常なら手作業で何時間も要する徹底的な調査
競合やベンダーの選択肢を評価するような比較分析
検証できる引用付きのレポート

Research は次のようなタスクに最適です:

市場分析と競合リサーチ
チームオフサイトやプロダクトローンチのような複雑なプロジェクトの計画
メール、カレンダー、ドキュメントからの情報の統合
複数のソースから引き出す技術ドキュメントの作成
最新の検証済み情報を必要とするブリーフィングの準備

代わりに Web 検索を検討すべきとき:

すばやい具体的な事実が必要なとき（今日の株価や会社の住所など）
答えに1つか2つのソースしか必要ないとき
包括性よりもスピードが重要なとき

代わりに拡張思考を検討すべきとき:

外部情報を必要としない複雑な問題について深い推論が必要なとき
数学の問題、コードのデバッグ、論理的分析に取り組んでいるとき
答えが情報の収集ではなく問題を推論することから得られるとき

代わりに Enterprise Search を検討すべきとき:

組織の内部ナレッジ（ドキュメント、Slack スレッド、メール、会議メモ）から引き出す答えが必要なとき
オンボーディング中で、会社が何かをどう扱っているか（ポリシー、プロセス、過去の決定など）をすばやく見つけたいとき
公開 Web ではなく、自社に固有の質問をしているとき
Research の仕組み

Research を有効にすると、単純な Web 検索をはるかに超えるエージェント型の多段階プロセスを起動します。Claude は、すでに見つけたものに基づいて次に何を検索するかを自律的に決定し、手がかりを追い、ギャップを埋めます。各ステップを指示する必要はありません。

### レッスン 12: Other ways to work with Claude

**URL:** <https://anthropic.skilljar.com/claude-101/383402>  

所要時間の目安: 10分

**学習目標**

このレッスンを終えると、以下ができるようになります:

Claude Code、Claude for Slack、Claude for Excel、Claude for PowerPoint、Claude for Chrome を含む追加の Claude 製品をいつ使うべきかを理解する

このコースの冒頭で述べたように、Claude はインテリジェンス（知性）です。Claude.ai はそれと協働する1つの方法に過ぎません。

Claude は、あなたがすでに働いている場所であなたを支えるよう設計された、いくつかの専門ツールでも利用できます。このレッスンでは、Claude と協働する5つの追加の方法を紹介します。それぞれが特定のワークフローやユースケースに合わせて調整されています。

Claude Code

Claude Code は、あなたが働く場所（ターミナル、IDE、ブラウザ、さらには Slack）で動作するエージェント型のコーディングツールです。コードベースを理解し、コマンドを実行し、自然言語を通じて開発ワークフロー全体を処理します。

Claude Code を使うべきとき:

必要なものを平易な英語で説明して機能を構築し、Claude にコードを書かせ、テストを実行させ、コミットを作成させたいとき
エラーメッセージを貼り付けて問題をデバッグし、Claude にコードベースを分析させて問題を特定・修正させたいとき
馴染みのないコードベースをナビゲートしていて、さまざまな部分がどう連携しているかについて質問したいとき
リントエラーの修正、マージコンフリクトの解決、リリースノートの作成のような面倒なタスクを自動化したいとき
別のインターフェースに切り替えるのではなく、既存の IDE や開発ツールと並んでターミナルで作業したいとき
Claude in Slack

Claude は Slack と直接統合され、チャンネルやスレッドで助けを得たり、Slack のコンテキストを Claude の会話に持ち込んだりできます。

Claude in Slack を使うべきとき:

Slack を離れることなく、メッセージへの返信を起草したり、長いスレッドを要約したり、複雑な議論を分解したりしたいとき
Claude にワークスペースから関連する会話や共有ドキュメントをまとめさせて、会議に備えたいとき
新しいチームにオンボーディング中で、チャンネル履歴をレビューして進行中のプロジェクトを理解する助けが欲しいとき
バグレポートや機能の議論から直接コーディングタスクを引き継ぎたいとき。@Claude をタグ付けするだけで、周囲のコンテキストを使って Claude Code セッションを立ち上げられます
会話の最中に、業界トレンド、技術的概念、会社情報についての答えをすばやく得たいとき
Claude for Excel

Claude for Excel は、サイドバーを通じて Claude を Microsoft Excel に直接持ち込み、会話を通じてスプレッドシートを分析・理解・変更できるようにします。

Claude for Excel を使うべきとき:

複雑な複数タブのワークブックで作業していて、特定の数式や計算フローがシートをまたいでどう機能するかを理解したいとき
数式の依存関係や関係性を保ちながら、モデル全体の前提や入力を更新する必要があるとき
#REF!、#VALUE!、循環参照のようなスプレッドシートのエラーをデバッグしていて、Claude にそれらを発生源まで追跡させ修正案を提案させたいとき
適切な数式構造を維持しながら、新しいスプレッドシートを作成したり既存のテンプレートにデータを入力したりしたいとき
データを可視化するためにピボットテーブルやチャートをすばやく構築する必要があるとき
Claude for PowerPoint

Claude for PowerPoint は、Claude をサイドバーとして Microsoft PowerPoint に持ち込み、既存のテンプレートやブランドスタイリングをそのまま保ちながら、会話を通じてプレゼンテーションを起草・編集・再構成できるようにします。

Claude for PowerPoint を使うべきとき:

各スライドを手作業で作ることなく、アウトライン、ドキュメント、メモのセットを初稿のスライドデッキに変えたいとき
スライドのコピーを書き直したり引き締めたりする必要があるとき。箇条書きを短くしたり、スピーカーノートを追加したり、特定のオーディエンス向けにトーンを調整したりするときです
既存のデッキを再構成していて、セクションの並べ替え、内容が詰まったスライドの分割、重複するスライドの統合の手助けが欲しいとき
各スライドを手作業で修正することなく、デッキ全体で一貫したフォーマット（タイトル、箇条書きのスタイル、レイアウト）を適用したいとき
スライドのためのすばやい視覚的提案が欲しいとき。たとえば、伝えようとしているポイントにどのレイアウトやチャートタイプが最も合うかなどです
Claude for Chrome

Claude for Chrome は、Google Chrome に Claude をサイドバーとして追加するブラウザ拡張機能です。あなたが取り組んでいるものを観察し、ブラウザ内で直接アクションを取れます。

Claude for Chrome を使うべきとき:

ブラウジング中に記事、研究論文、Web ページを要約したいとき
メールの返信を起草したり受信トレイを管理したりする助けが必要なとき
繰り返しのフォームに記入していて、そのプロセスを自動化したいとき
各ステップを手作業でクリックすることなく、Web サイトの機能をテストしたり多段階のワークフローをナビゲートしたりしたいとき
タブやタスクの間を移動してもコンテキストを維持するブラウジングアシスタントが必要なとき。これにより、ニッチな社内ツール、CRM、ダッシュボードからコンテキストを引き出すのに最適になります

重要な注意: Claude for Chrome は現在リサーチプレビュー中です。Anthropic は、信頼できる Web サイトでの低リスクのタスクに使うことを推奨しています。この拡張機能は、購入や個人データの共有のような高リスクのアクションを取る前に許可を求めます。
