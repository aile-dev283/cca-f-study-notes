## Part 1: Effective Harnesses for Long-Running Agents

### 核心的な課題

「長期実行エージェントの根本的な課題は、離散的なセッションで作業しなければならず、新しいセッションはそれ以前の記憶なしに始まる」

= シフト制で働く開発チームに例えられる状態。

### 2つの失敗パターン

| パターン | 説明 |
|---------|------|
| 過度な一括処理 | エージェントが過剰に多くの機能を同時に実装 → コンテキスト枯渇 → 半完成の機能を残す |
| 早期完了宣言 | 後続セッションで進捗があると判断して作業を終了と宣言 |

> **Compaction だけでは不十分**。コンテキスト管理機能だけでは Claude Opus 4.5 でも長期タスク完遂が困難。構造的指示と環境設計が必須。

### 解決策：初期化エージェント + コーディングエージェントの分離

#### 初期化エージェント（初回セッション）
- `init.sh`：開発環境の起動を自動化
- `claude-progress.txt`：エージェント作業の履歴ログ
- 初期 git commit：ファイル追加の明確な記録

#### コーディングエージェント（以降のセッション）
- 単一機能に限定した段階的進捗
- 構造化された更新と git コミット
- セッション終了時の環境クリーンアップ

### 機能リストの設計

200 以上の詳細な機能仕様を JSON 形式で作成。各機能に：
- カテゴリ分類
- 詳細な説明とステップ
- `"passes"` フィールド（true/false）

> **JSON 形式を採用する理由**：マークダウンより改変されにくく、不適切な変更を防止できる。「テストを削除・編集することは受け入れられない」

### セッション開始プロトコル

1. 作業ディレクトリの確認（`pwd`）
2. git ログと進捗ファイル読み込み
3. 機能リストから優先度最高の未実装機能を選択
4. `init.sh` 実行による開発サーバー起動
5. 基本的なエンドツーエンド機能検証

### 失敗パターン対応表

| 問題 | 初期化エージェント対応 | コーディングエージェント対応 |
|------|----------------------|--------------------------|
| 早期完了宣言 | 機能リストファイル作成 | 単一機能フォーカス実装 |
| 不完全な環境状態 | git + 進捗ノート | セッション開始時検証と終了時更新 |
| 不十分なテスト | 機能リスト定義 | 自己検証と慎重な完了判定 |
| アプリケーション起動困難 | init.sh 作成 | セッション開始時実行 |

### 実装からの学習

1. 人間エンジニアの実務慣行が有効な指針
2. JSON 形式はマークダウンより改変されにくい
3. 明示的なテストツール（ブラウザ自動化）が必須
4. 段階的アプローチが効率性向上

## Part 2: Equipping Agents with Agent Skills

### Agent Skills の概要（2025年10月16日発表）

「指示・スクリプト・リソースの整理されたフォルダで、エージェントが動的に発見・ロードできる」

**目的**：一般的なエージェント機能を特定分野に特化させる。

### SKILL.md の基本構造

```markdown
---
name: pr-review
description: Review a pull request for correctness and style
---

# PR Review Skill
[指示の詳細...]
```

### 段階的情報開示（Progressive Disclosure）

「よく整理されたマニュアルが目次から始まるように」：

| 段階 | 読み込み内容 |
|-----|-----------|
| 第 1 段階 | メタデータのみ（name + description） |
| 第 2 段階 | SKILL.md の完全な内容 |
| 第 3 段階以降 | `reference.md`・`forms.md` などの関連ファイル |

→ コンテキストウィンドウを効率的に使用。

### MCP との関係

- MCP サーバーを**補完**する：「より複雑なワークフローをエージェントに教える」
- MCP は外部ツール統合、Skills は内部ワークフロー指示

### セキュリティ警告

- 「悪意ある Skills は脆弱性を導入する可能性がある」
- **trusted source からのみインストール**
- インストール前の監査必須：
  - バンドルされたコードと依存関係
  - 外部ネットワーク接続の指示

## 試験対策：判断軸

| 状況 | 正解寄り | 罠 |
|------|---------|-----|
| 長期タスクのセッション管理 | init.sh + progress log + 単一機能フォーカス | 高レベルプロンプトだけで長期タスクを一発依頼 |
| セッション間継続 | git + structured progress file + memory tool | 自然文の要約だけに頼る |
| compaction | 必要だが十分ではない（構造的設計が必須） | compaction だけで長期タスクを完遂しようとする |
| Skills のセキュリティ | trusted source のみ・内容 audit | marketplace の他人の skill を無検査で入れる |
| Skills vs CLAUDE.md | 繰り返しワークフロー → Skill、常時必要な知識 → CLAUDE.md | 何でも CLAUDE.md に詰める |
