# Developer Productivity Scenario (S2/S4)

このワークスペースはチームが Claude Code を開発生産性向上に使うシナリオを模倣する。

## チーム規約

- 言語: Python 3.12+（型ヒント必須）
- テスト: pytest、カバレッジ 80% 以上を目標
- コミット: Conventional Commits 形式（`feat:` / `fix:` / `refactor:`）
- PR サイズ: 変更行数 400 行以内を目安

## Claude への指示

- リファクタリングを提案する前に **なぜ** そうするかを説明すること
- 6ヶ月以上変更されていないコードを削除する前に確認を取ること
- コード変更後は必ずテストを実行: `pytest -x`
- 未知のレガシーコードは: 読む → 要約する → 編集する の順で進むこと

## 禁止事項

- `legacy/` ディレクトリは明示的な確認なしに変更しないこと
- `pyproject.toml` を更新せずに依存パッケージを追加しないこと
- テストをスキップ（`pytest -x` の `-x` を外す等）しないこと
